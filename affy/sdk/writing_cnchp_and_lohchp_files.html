Writing Command Console CNCHP and LOHCHP Files

 

 

 

INTRODUCTION

The Command Console CNCHP and LOHCHP files are used to store Copy Number and Loss of Heterozygosity (LOH) results. Copy Number results generated by the CN4 algorithm are stored in CNCHP files. LOHCHP files retain results generated by the CN4 algorithm. Copy Number and Loss of Heterozygoisity results generated by the CN5 algorithm are combined and stored in CNCHP files.

The data objects used to create CNCHP and LOHCHP files are the same as those used to create CHP files storing genotyping results generated by the BRLMM and Birdseed algorithms. (See How to write a Command Console CHP file for more information on  writing files containing data generated by BRLMM and Birdseed algorithms.) These data objects can store a variety of data in a matrix format (rows and column of values).

This document will describe how to use the Fusion SDK to write data to Command Console CNCHP and LOHCHP files.

SDK Objects

Results generated by Copy Number and LOH analyses are retained in a file object called CHPMultiDataData object.

Command Console CNCHP and LOHCHP files can be created in two steps using the buffer writer technique. The CHP file header is created first. The remainder of the file will be created at a later time. The workflow below describes this two step approach.

The main top level SDK objects that are required are:

  • affymetrix_calvin_io::CHPMultiDataFileWriter - This object provides the writing capabilities.
  • affymetrix_calvin_io::CHPMultiDataFileBufferWriter - This object provides the capability to set and update results data
  • affymetrix_calvin_io::CHPMultiDataData - This object provides the storage for the CHP file results.

WORKFLOW

The workflow is as follows:

1.     Create a CHPMultiDataData object in the name of the output CHP file into the constructor. The extension of the file name should be as follows:

o    "CNCHP" for Copy Number results from the CN4 algorithm

o    "LOHCHP" for LOH results from the CN4 algorithm

o    "CNCHP" for Copy Number and LOH results generated by the CN5 algorithm

2.     Set the header information. This is done using the following functions:

CHPMultiDataData.SetAlgName - to set the name of the algorithm.

CHPMultiDataData.SetAlgVersion - to set the version number of the algorithm.

CHPMultiDataData.SetArrayType - to set the name of the probe array type (e.g. Genome-Wide  Human SNP Array 6.0)

CHPMultiDataData.SetEntryCount - to set the data shown below.  This function is called for each analysis type to be stored in the file.

a.     The analysis type for both Copy Number and LOH analysis results will be affymetrix_calvin_io::MultiDataType::CopyNumberMultiDataType

b.     The number of probeset names to be stored

c.     The maximum length of the names of the probesets that will be stored in the file.

d.     A vector of affymetrix_calvin_io::ColumnInfo objects containing the definition of additional result columns to include in the file for the analysis.

CHPMultiDataData.AddAlgParams - to add a list of algorithm parameters to the file header as name/value/type attributes . 

CHPMultiDataData.AddSummaryParams - to add a list summary statistics as name/value/type attributes (e.g. CNAT4 algorithm adds the IQR values to this chip summary section).

Affymetrix recommends you store information about the program in the header of CNCHP and LOHCHP files. The program information should include the company or institution writing the software, the name of the program and the version of the program. These should be stored in the name/value attribute list of the file header. The attributes names should be:

o    "program-name" to identify the name of the program

o    "program-version" to identify the version of the program

o    "program-company" to identify the name of the company or institution writing the software.

Use the CHPMultiDataData::GetGenericData().Header().GetGenericDataHdr()->AddNameValParam() function to add these and other parameters to the file header.

3.     Add to the header information about the parent file or files. The Command Console file has the ability to store information about the parent CEL file or files. For each parent file:

a.     Create a GenericDataHeader object and set the attributes:

GenericDataHeader.SetFileId - this is a globally unique identifier for the file.

GenericDataHeader.SetFileTypeId - this identifies the type of file. Use the constant affymetrix_calvin_data::INTENSITY_DATA_TYPE to identify a CEL file.

b.     Add parameters using the GenericDataHeader.AddNameValParam function.

c.     Add the GenericDataHeader object to the CHPMultiDataData object using the CHPMultiDataData.GetFileHeader()->GetGenericDataHdr()->AddParent() function.

4.     Set the file type identifier using the CHPMultiDataData.GenericData.Header().GetGenericDataHdr()->SetFileTypeId() function. This identifies the type of data to be stored in the file.

5.     CHPMultiDataFileWriter object passing in the CHPMultiDataData object into the constructor.  This will create the file with the header information.

6.     Create the body of the file with results using a file buffer writer object as described below.

a.     Create a CHPMultiDataFileBufferWriter.  Use the CHPMultiDataFileBufferWriter.Initialize function to set the name of the CNCHP  or LOHCHP file, the type of result data to be stored and the maximum length of the probeset names to be stored. 

b.     To store the analysis results, instantiate a ProbeSetMultiData entry object that corresponds to the MultiDataType used in the above step.  Loop through the results completing the following steps:

                                      i.        Set the name data member of the entry object with that of the probeset.

                                     ii.        Set the results for the default columns of the entry object

                                    iii.        Add the results for any additional columns defined when the file header was created to the metrics vector of the entry object.

                                    iv.        Write the results using the GenotypEntryBufferWriter.WriteMulitDataGenotypeEntry function.

c.     Use the GenotypeEntryBufferWriter.FlushBuffer function to flush the buffer and close the file.

Requirements And Recommendations

The code should be placed within a try/catch statement as the writers will throw exceptions upon errors.


It is highly encouraged to add the name of your company or institution to the algorithm parameters section of the CNCHP and LOHCHP files. This way you can track those files created by your software.
