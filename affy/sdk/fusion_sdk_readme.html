AFFYMETRIX DEVELOPER NETWORK

 

 

Affymetrix Fusion SDK - Release 1.0.10a (December 2007)

 

 

 

INTRODUCTION

The Fusion SDK provides ANSI C++ source code for parsing data files generated and managed by the GCOS 1.x software and the new Affymetrix GeneChip Command Console software (newsletters and recorded WebTalks are available on the Affymetrix DevNet web site for more information on the Command Console software). Using the Fusion SDK provides backwards and forward flexibility by providing the ability to parse both types of data files.

The SDK was developed using the ANSI C++ language and tested using the Microsoft Visual C++ 2005 compiler on a Windows XP system, G++ 3.3 compiler on a Apple OSX 10.3 system and G++ 3.3 on a Sun Solaris system. The Java code in the Fusion SDK required JRE 1.5 or above.

The documentation provided and source code in the Fusion SDK contains references to the project code named "Calvin". This code name is used to refer to the Affymetrix GeneChip Command Console software.

The source code in the Fusion SDK is licensed under the GNU Lesser General Public License (LGPL).

RELEASE NOTES

Version 1.0 Updates after pre-release

  • The Command Console generic data file format has been modified to include the addition of column names for the columns in a data set.
  • Support for resequencing CHP files was added.
  • Support for tiling CHP files was added.
  • A flag was added to indicate if absolute or comparison expression values are stored in the CHP file.
  • The method for reading CHP files now uses the static method FusionCHPDataReg::CreateObject. This method determines the type of CHP file (expression, genotyping, tiling, etc.) and returns a pointer to the specific CHP data file object.
  • The parameter names in the CHP file for algorithm and chip summary parameters have been changed to be more consistent with the CEL file parameter names.
  • CHP and Array file writing classes have been added.

Version 1.0.03 Update (December 2005)

  • Updates to the pure Java implementation. Souce code, javadocs and a pre-built JAR file are included.
  • New COM implementation (beta). This is provided as C++ code and a pre-built COM DLL.
  • Additional interfaces to the Fusion CEL and CHP layer to expose the file identifier and the generic data object for "Command Console" files.

Version 1.0.04 Updates (February 2006)

  • Added a method to update the data in a CHP file after written (DataSetUpdater class).
  • Performance improvements were added to the CHP file writers.
  • Added class to read a CHP file with only a probe set name and signal value (FusionCHPSignalData class).
  • Added a method to retrieve the GenericData object from the fusion layer in the CEL and CHP parsers.
  • Added a method to retrieve the file identifier from the fusion layer in the CEL and CHP parsers.
  • Added a method to add a floating point and integer value to the chip summary section of the expression CHP object.
  • Added the library package name, media file and guid to the array set file DTD. The new DTD is documented in the file formats document.
  • Fixed compile errors with the Mac platform.
  • Bug fix with the CHP parser when reading the various types of CHP files.
  • Bug fix with the GenericDataHeaderUpdater class on the Mac platform.

Version 1.0.05 Updates (March 2006)

  • Bug fix with the FusionExpressionProbeSetResults::GetNumUsedPairs() function. The function was returning the number of pairs.
  • Documentation update for writing CHP files. The document was updated to reflex new functions that must be called when writing CHP files. The functions "CHPFileWriter::SeekToDataSet" and "CHPFileWriter::SeekToBgSet" were added in version 1.04 of the Fusion SDK to improve performance of writing CHP files. This update is with the documentation only.
  • Changed the method for creating CHP files to take in the maximum length of a probe set name. Previously the maximum length was hard coded to 32 characters.
  • Changed the probe set name type from wstring (16 bit) to string (8 bit).
  • Added support to compile the code under the Solaris platform using G++ 3.3.
  • Updated the Java samples to use the pure Java implementation rather than the SWIG generated wrappers.
  • The CustomerBarcode entry from the array set file DTD was removed.

Version 1.0.06 Updates (May 2006)

  • Bug fix in the Java CEL parser for Command Console CEL files. The identifier for the number of rows and columns was incorrect.
  • Added interfaces to the CEL parsers (C++ and Java) to retrieve the library package and master file name. These are new items being captured by the Command Console software.
  • Replaced the use of the WIN32 macro with the _MSC_VER macro. This is to isolate code specific to the Microsoft compiler.

Version 1.0.07 Updates (July 2006)

  • The classes for reading Expression Console CHP files (non-MAS5 algorithm generated CHP files) has changed. The update was to change the name of the classes to use "Quantification" instead of "Signal". The new classes are:
    • FusionCHPQuantificationData (replacing "FusionCHPSignalData")
    • FusionCHPQuantificationDetectionData (new class for Exon CHP files generated from the Expression Console software)

Version 1.0.08 Updates (September 2006)

  • Modified the DataSet class to handle retrieval of more than 200MB of data.
  • Bug fix with reading Expression Console generated CHP files (with was released as patch 1.0.7.1).

Version 1.0.09 Updates (February 2007)

  • Added support for new genotyping format CHP files ("multi-data" CHP file).
  • Added support for more probe set  types in the CDF file
  • Changed the method of reading CDF file to reduce the amount of memory required. The new method does not memory map the entire file.

Version 1.0.10 Updates (November 2007)

  • Added support for copy number format CHP files (using the "multi-data" CHP file format). These have CNCHP and LOHCHP extensions.
  • Updated the Java source to utilize generics. The Java code now requires JRE1.5 and above.
  • Added support in the Java code for writing CHP files.
  • Added example C++ code for writing expression and genotyping CHP files.
  • Removed the SWIG wrappers. 

Version 1.0.10a Updates (December 2007)

  • Modified the Java source to exclude colons from Affymetrix GUIDS.
  • Changed the Unicode encoding scheme for the Java Array writer to UTF-16.
  • Modification to the Java source to be JRE1.5 compliant.

Version 1.0.10b Updates (January 2008)

  • Bug fix in Java class for handling C++ unsigned integers.
  • Updated Ant build file for Java SDK.

 

GETTING STARTED

Three tutorials are provided to document using the fusion classes to read and write the data files. This documentation and the samples programs supplied is the easiest way to get started using the Fusion SDK.  C++ and Java sample programs in the affy/sdk/parser_examples/  directory provide examples for reading CEL, CDF and CHP files.  Sample C++ code for writing expression and genotyping CHP files can be found in the affy/sdk/calvin_files/converters/ directory.

Tutorials include:

  • How to read a Command Console or GCOS data file
  • How to write a Command Console CHP file
  • How to write Command Console CNCHP and LOHCHP files
  • How to write a Command Console Array file

COMPILING THE CODE

There are several macros that may need to be defined, using the "-D" compiler switch, to enable compilation and execution on specific operating systems. These are:

Macro Definition

Description

IS_BIG_ENDIAN

Define this value if the machine represents its data in big ENDIAN format. If the machine is a little ENDIAN machine (such as an Intel system) then omit this macro.

_INCLUDE_UNISTD_HEADER_

Define this value if the UNISTD.H header file is used by the C++ compiler on your machine.

WIN32

Define this value if using the MS VC++ compilers.

_USE_MEM_MAPPING_

This value is used to indicate if memory mapping should be used by the GCOS 1.x BAR, BPMAP and CDF file parsers. Memory mapping is automatically used for CEL file parsing and by the Command Console file parsers.

3RD PARTY TOOLS

XERCES

The SDK uses the XERCES library for parsing XML files. Source code for this library is not included as part of the Fusion SDK.  Current Command Console formatted XML files are:

  • Array Files
  • Templates Files
  • DTT 1.1 XML MAGE Files

The XERCES library is available at: http://xml.apache.org/xerces-c/. You must download and build this library to use the XML parsing objects of the Fusion SDK. The Fusion SDK was tested with XERCES version 2.7.

CPPUNIT

The Fusion SDK contains a set of unit test cases to test each class within the SDK. These unit tests were developed using the CPPUNIT Framework version 1.10.2. Test case source and Visual Studio Projects are located in the "affy/sdk/calvin_files/*/test" folders. Each Visual Studio Project (located in the test directory - e.g. affy/sdk/calvin_files/array/test) will compile, build and execute the test cases. The test cases will run giving a "." for each test when it completes. If a "F" is displayed instead of an ".", then that test case failed. When all the test cases were attempted and failed or passed, those that failed will be indicated with the source line and file where the error occurred. The number of passed and failed test cases will be indicated as well. If all test cases pass, then an "OK" is indicated along with the test case count. The test cases may also be performed inside the development environment in debug mode. Stepping through code during debug is also supported (Visual Studio).

Test data is provided in the data folder one level up from the test folders - e.g. affy/sdk/calvin_files/parsers/data. An additional test data folder is provided for larger files and that is in common to both GCOS 1.x File SDK and Fusion. This test data is located in the "affy/sdk/TestDataFiles" folder.

Installation Directories

The 3rd party tools must be installed at the same level as "affy" directory. The folder names must be "xerces-c" and "cppunit". These names and relative locations to the affy directory are assumed within each Visual Studio project.

Example:

  • /development/affy/sdk/...
  • /development/cppunit
  • /development/xerces-c
